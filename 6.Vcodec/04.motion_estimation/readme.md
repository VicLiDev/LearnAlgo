# 运动估计 (Motion Estimation)

## 1. 简介

运动估计是视频压缩的核心技术，通过在参考帧中搜索匹配块来消除时间冗余。

## 2. 块匹配算法

### 2.1 基本原理

```
当前块位置: (x, y)
搜索范围: [x-dR, x+dR] × [y-dR, y+dR]

找最优匹配位置 (mx, my) 使 SAD/COST 最小
```

### 2.2 匹配准则

| 准则 | 公式 | 计算量 |
|------|------|--------|
| SAD | Σ \|curr - ref\| | 最小 (无乘法) |
| SSD | Σ (curr - ref)² | 中等 |
| SATD | Hadamard变换后SAD | 较大 |
| MAD | SAD / (N×N) | 最小 |

### 2.3 搜索算法

| 算法 | 搜索点数 | 特点 |
|------|----------|------|
| 全搜索 (FS) | (2dR+1)² | 最优但慢 |
| 三步搜索 (TSS) | ~25 | 经典快速算法 |
| 菱形搜索 (DS) | ~15 | H.264常用 |
| 六边形搜索 | ~12 | 更快 |
| UMHexagonS | ~30 | H.264高速模式 |

## 3. 菱形搜索 (Diamond Search)

### 3.1 搜索模板

```
大菱形:          小菱形:
    ○               ○
  ○ ● ○             ●
    ○               ○
```

### 3.2 流程

```
1. 以原点为中心，使用大菱形模板
2. 找最小SAD点
3. 如果最小点在中心，转到小菱形
4. 否则以最小点为中心继续大菱形
5. 小菱形确定最终MV
```

## 4. 运动矢量预测

### 4.1 空域预测

```
使用相邻块的MV预测当前块MV:

    A  B  C
       ↓
    [当前]

预测MV = median(MV_A, MV_B, MV_C)
```

### 4.2 时域预测

```
使用前一帧同位置块的MV:

预测MV = collocated_MV × (curr_poc - ref_poc) / (col_poc - col_ref_poc)
```

### 4.3 MV编码

```
MVD = 实际MV - 预测MV

只编码残差MVD，节省比特
```

## 5. 亚像素运动估计

### 5.1 插值过程

```
整像素搜索 → 1/2像素搜索 → 1/4像素搜索

插值滤波器:
- 半像素: 6抽头滤波器
- 1/4像素: 双线性插值
```

### 5.2 计算复杂度

```
整像素: 1×
1/2像素: 2× (水平和垂直)
1/4像素: 2× (在1/2基础上)

总复杂度约4倍整像素搜索
```

## 6. 特殊模式

### 6.1 Skip/Direct模式

- 直接使用预测MV，不编码MVD
- 适合平坦/背景区域

### 6.2 Merge模式

- 从候选列表选择MV
- 只编码候选索引

## 7. 率失真优化 (RDO)

### 7.1 模式选择

```
Cost = D + λ × R

D: 失真 (SSE/SAD)
R: 编码比特数
λ: 拉格朗日乘子

选择 Cost 最小的模式
```

### 7.2 运动估计RDO

```
整像素: Cost = SATD + λ × bits(MVD)
亚像素: Cost = SATD + λ × bits(MVD + subpel_bits)
```
