# 霍夫投票 (Hough Voting)

## 1. 简介

霍夫投票（Hough Voting）是一种基于投票机制的参数估计方法，通过将图像空间中的点
转换到参数空间，实现对几何形状（直线、圆、椭圆等）的鲁棒检测。该方法对噪声和部分
遮挡具有较好的鲁棒性，广泛应用于计算机视觉、图像处理、3D物体检测等领域。

### 1.1 核心思想

**对偶性原理**：图像空间中的一个点对应参数空间中的一条曲线（或曲面），参数空间中的
一个点对应图像空间中的一条曲线（或形状）。

- **图像空间**：包含边缘点的2D/3D空间
- **参数空间**（累加器空间）：描述目标形状参数的空间
- **投票机制**：每个边缘点为其可能所属的形状参数投票

### 1.2 应用场景

| 应用领域   | 具体任务                   |
|------------|----------------------------|
| 图像处理   | 直线检测、圆检测、椭圆检测 |
| 工业检测   | 零件定位、缺陷检测         |
| 自动驾驶   | 车道线检测、交通标志识别   |
| 机器人视觉 | 物体位姿估计、抓取点定位   |
| 3D视觉     | 点云中的形状检测、物体识别 |

## 2. 霍夫变换基础

### 2.1 标准霍夫变换（SHT）

#### 直线检测

**参数化表示**：
```
ρ = x·cos(θ) + y·sin(θ)

其中：
- ρ：原点到直线的垂直距离
- θ：垂线与x轴的夹角 [0, π)
```

**算法流程**：
```
输入: 边缘图像，θ分辨率dθ，ρ分辨率dρ
输出: 检测到的直线参数 [(ρ₁, θ₁), (ρ₂, θ₂), ...]

1. 初始化累加器数组 A(ρ, θ) = 0
2. 确定参数空间范围：
   - θ ∈ [0, π)
   - ρ ∈ [-√(W²+H²), √(W²+H²)]
3. 对每个边缘点 (x, y):
   对每个 θ ∈ [0, π) 步长 dθ:
     计算 ρ = x·cos(θ) + y·sin(θ)
     A(ρ, θ) += 1
4. 检测累加器中的局部极大值
5. 返回超过阈值的 (ρ, θ) 作为检测到的直线
```

**复杂度分析**：
- 时间复杂度：O(n·n_θ)，其中n是边缘点数，n_θ是θ的离散化数量
- 空间复杂度：O(n_ρ·n_θ)

### 2.2 概率霍夫变换（PHT）

通过随机采样减少计算量：

```
1. 随机采样边缘点子集（如25%）
2. 对采样点执行标准霍夫变换
3. 当检测到直线时，利用所有边缘点验证
```

**优点**：计算量大幅减少
**缺点**：可能漏检某些直线

### 2.3 累进概率霍夫变换（PPHT）

结合随机采样和提前终止策略：

```
1. 随机采样边缘点
2. 当某条直线的票数达到阈值时：
   - 移除该直线上的所有边缘点
   - 重置对应的累加器
3. 重复直到：
   - 采样点数达到最大值，或
   - 没有足够的边缘点形成直线
```

**优点**：
- 计算效率高
- 减少虚假直线
- 适合实时应用

## 3. 广义霍夫变换（GHT）

### 3.1 任意形状检测

对于无法用简单参数方程描述的形状，使用广义霍夫变换：

**R-table 构建**：

```
1. 选择参考点 c（通常是形状质心）
2. 对每个边缘点 pᵢ：
   - 计算梯度方向 φᵢ = atan2(∂I/∂y, ∂I/∂x)
   - 计算相对向量 rᵢ = c - pᵢ
   - 将 (rᵢ, φᵢ) 存入 R-table[φᵢ]
```

**检测过程**：

```
1. 对每个边缘点 pᵢ：
   - 计算梯度方向 φᵢ
   - 从 R-table[φᵢ] 获取所有可能的 r
   - 对每个候选参考点 c = pᵢ + r 投票
2. 累加器中的峰值即为形状位置
```

### 3.2 数学描述

设形状有M个边缘点，R-table包含K个量化方向：

**投票方程**：
```
A(x, y) = Σᵢ Σⱼ δ(φ(pᵢ), φⱼ) · δ(pᵢ + rⱼ, (x,y))

其中：
- A(x, y)：位置(x,y)的票数
- φ(pᵢ)：点pᵢ的梯度方向
- rⱼ：R-table中对应φⱼ的相对向量
```

## 4. 3D霍夫投票

### 4.1 点云中的平面检测

**参数化**：
```
ρ = n·p = nₓ·x + nᵧ·y + n_z·z

参数空间： (nₓ, nᵧ, n_z, ρ)
约束： ||n|| = 1
```

**降维处理**：
使用球坐标表示法向量：
```
nₓ = sin(θ)·cos(φ)
nᵧ = sin(θ)·sin(φ)
n_z = cos(θ)

参数空间：(θ, φ, ρ)
```

### 4.2 3D物体检测（基于霍夫森林）

**霍夫森林（Hough Forest）**：使用随机森林学习从局部特征到物体中心的映射。

```
训练阶段：
1. 提取正样本的局部特征（如SIFT, SURF）
2. 记录特征相对于物体中心的偏移
3. 训练随机森林，叶子节点存储偏移向量

检测阶段：
1. 提取测试图像的局部特征
2. 通过森林获取每个特征的偏移预测
3. 在Hough空间投票
4. 峰值位置为物体中心
```

## 5. 霍夫圆检测

### 5.1 标准参数化

```
(x - a)² + (y - b)² = r²

参数空间：(a, b, r)
- (a, b)：圆心坐标
- r：半径
```

### 5.2 使用梯度信息加速

**2-1 Hough变换**：

利用边缘梯度方向减少搜索空间：

```
对每个边缘点 (x, y) 和梯度方向 (gₓ, gᵧ)：
  对每个候选半径 r：
    圆心候选 = (x, y) ± r·(gₓ, gᵧ) / ||g||
    在圆心位置投票
```

**优点**：
- 计算复杂度从O(n·nₐ·n_b·n_r)降至O(n·n_r)
- 减少虚假圆检测

### 5.3 自适应半径检测

当半径未知时：

```
方法1：多尺度检测
  对每个候选半径 r ∈ [r_min, r_max] 执行检测

方法2：梯度幅值约束
  利用边缘强度估计可能的半径范围
```

## 6. 算法优化

### 6.1 累加器设计

| 方法       | 描述             | 适用场景   |
|------------|------------------|------------|
| 固定网格   | 均匀量化参数空间 | 标准检测   |
| 自适应网格 | 高分辨率区域细化 | 高精度要求 |
| 多分辨率   | 金字塔式粗到细   | 大范围搜索 |
| 连续域     | 使用核密度估计   | 亚像素精度 |

### 6.2 峰值检测

**局部极大值检测**：

```
1. 找到全局最大值
2. 在邻域内抑制（非极大值抑制）
3. 重复直到所有峰值低于阈值
```

**改进方法**：

```
方法1：阈值滞后
  - 高阈值确定候选峰值
  - 低阈值扩展峰值区域

方法2：均值漂移（Mean Shift）
  - 从每个候选点开始
  - 沿密度梯度方向迭代
  - 收敛到局部密度极大值

方法3：EM算法
  - 将峰值检测建模为混合模型
  - 迭代估计各分量的参数
```

### 6.3 抗噪策略

**问题**：噪声和边缘断裂会导致虚假峰值

**解决方案**：

```
1. 边缘预处理
   - 形态学操作连接断裂边缘
   - 边缘平滑减少噪声

2. 投票权重
   - 根据边缘强度加权投票
   - 根据梯度一致性加权

3. 后处理验证
   - 验证检测形状的完整性
   - 移除支持点过少的候选
```

## 7. 实现细节

### 7.1 离散化分辨率选择

**角度分辨率 dθ**：
```
经验公式：dθ ≈ π / (W + H)

或根据应用需求：
- 精度优先：dθ = π/180 (1度)
- 速度优先：dθ = π/90 (2度)
```

**距离分辨率 dρ**：
```
通常：dρ = 1 (像素)
高精度：dρ = 0.5 (亚像素)
```

### 7.2 阈值设置

**投票阈值**：
```
最小票数 = max(
  边缘点数 × min_shape_ratio,
  absolute_min_threshold
)

其中 min_shape_ratio ∈ [0.05, 0.15]
```

**峰值检测阈值**：
```
方法1：固定阈值
  threshold = k × max_votes, k ∈ [0.3, 0.5]

方法2：自适应阈值
  threshold = mean + σ × std
```

### 7.3 参数空间范围

**直线检测**：
```
θ ∈ [0, π)
ρ ∈ [-D, D], D = √(W² + H²)
```

**圆检测**：
```
a ∈ [r, W-r]
b ∈ [r, H-r]
r ∈ [r_min, r_max]
```

## 8. 与其他算法比较

### 8.1 vs RANSAC

| 特性       | 霍夫变换   | RANSAC        |
|------------|------------|---------------|
| 原理       | 投票机制   | 随机采样      |
| 鲁棒性     | 对噪声鲁棒 | 对离群点鲁棒  |
| 计算复杂度 | O(n·n_θ)   | O(k·n_sample) |
| 多形状检测 | 天然支持   | 需要迭代移除  |
| 参数空间   | 高维时复杂 | 相对简单      |
| 实时性     | 可优化     | 较好          |

### 8.2 vs 最小二乘

| 特性       | 霍夫变换 | 最小二乘 |
|------------|----------|----------|
| 离群点处理 | 鲁棒     | 敏感     |
| 多模型     | 支持     | 不支持   |
| 参数估计   | 离散化   | 连续优化 |
| 计算量     | 较大     | 较小     |

## 9. 高级主题

### 9.1 霍夫变换加速

**GPU加速**：
```
- 每个边缘点由独立线程处理
- 原子操作更新累加器
- 并行峰值检测
```

**分层霍夫变换**：
```
1. 在低分辨率图像上粗略检测
2. 在候选区域高分辨率精确定位
3. 复杂度从O(n)降至O(log n)
```

### 9.2 形状上下文霍夫

结合形状上下文描述符：

```
1. 对每个边缘点计算形状上下文描述符
2. 使用描述符匹配进行加权投票
3. 提高对形变和遮挡的鲁棒性
```

### 9.3 隐式形状模型（ISM）

```
训练：
1. 检测训练图像中的兴趣点
2. 建立码本（codebook）
3. 存储每个码字相对于物体中心的偏移

检测：
1. 检测测试图像中的兴趣点
2. 匹配码本并获取偏移
3. 在Hough空间投票
4. 检测物体位置和尺度
```

## 11. 常见问题

### Q1: 如何选择累加器分辨率？

**A**: 分辨率越高，精度越高，但计算量和内存消耗也越大。
- 一般原则：分辨率应匹配数据精度
- 经验值：角度1-2度，距离1像素
- 高精度应用：角度0.5度，距离0.5像素

### Q2: 检测到很多虚假直线怎么办？

**A**:
1. 增加最小投票阈值
2. 增加最小直线长度约束
3. 应用非极大值抑制
4. 验证直线的边缘支持度

### Q3: 如何处理参数空间的高维问题？

**A**:
1. 使用梯度信息约束（如2-1 Hough）
2. 分层检测（先检测部分参数）
3. 随机霍夫变换
4. 使用霍夫森林等学习方法

### Q4: 霍夫变换能检测任意形状吗？

**A**: 可以，使用广义霍夫变换（GHT）：
- 训练阶段构建R-table
- 检测阶段使用R-table投票
- 支持平移、旋转、缩放变换

### Q5: 如何评估霍夫变换的性能？

**A**: 评估指标包括：
- 检测准确率（Precision）
- 检测召回率（Recall）
- 定位精度
- 计算时间
- 内存占用

## 12. 参数调优指南

### 12.1 直线检测参数

| 参数       | 推荐范围          | 影响             |
|------------|-------------------|------------------|
| 角度分辨率 | π/180 ~ π/90      | 精度 vs 速度     |
| 距离分辨率 | 0.5 ~ 2 像素      | 精度 vs 鲁棒性   |
| 投票阈值   | 边缘数×0.1~0.2    | 召回率 vs 精确率 |
| 最小长度   | 应用相关          | 过滤短直线       |
| 最大间隙   | 最小长度的0.3~0.5 | 连接断裂边缘     |

### 12.2 圆检测参数

| 参数      | 推荐范围          | 影响             |
|-----------|-------------------|------------------|
| 最小半径  | 应用相关          | 过滤小圆         |
| 最大半径  | 应用相关          | 限制搜索空间     |
| 圆心距离  | 期望半径的1.5~2倍 | 多圆检测         |
| 投票阈值  | 边缘数×0.3~0.5    | 召回率 vs 精确率 |
| Canny阈值 | 根据图像调整      | 边缘质量         |
