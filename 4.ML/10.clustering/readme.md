# 聚类算法 (Clustering Algorithms)

## 概述

聚类算法是无监督学习的一种，目标是将相似的数据点分组到同一个簇中，使簇内的点尽可能相似，簇间的点尽可能不同。

## 常用聚类算法

### 1. K-Means 聚类
- **原理**: 迭代地将数据点分配到最近的质心，然后重新计算质心
- **算法步骤**:
  1. 随机初始化K个质心
  2. 将每个点分配到最近的质心
  3. 重新计算每个簇的质心
  4. 重复步骤2-3直到收敛
- **优点**: 简单、快速、易于实现
- **缺点**: 需要预设K值、对初始值敏感、只能发现球形簇
- **适用场景**: 簇大小相近、形状为凸形的数据

### 2. 层次聚类 (Hierarchical Clustering)
- **原理**: 构建聚类的层次树结构
- **类型**:
  - 凝聚式：自底向上，每个点开始为一个簇，逐步合并
  - 分裂式：自顶向下，所有点开始为一个簇，逐步分裂
- **优点**: 不需要预设簇数、可以生成树状图
- **缺点**: 计算复杂度高、一旦合并/分裂不能撤销
- **适用场景**: 需要层次结构的场景

### 3. DBSCAN (Density-Based Spatial Clustering)
- **原理**: 基于密度的聚类，将高密度区域划分为簇
- **关键参数**:
  - ε (eps): 邻域半径
  - MinPts: 形成核心点的最小邻居数
- **优点**: 不需要预设簇数、可以发现任意形状的簇、能识别噪声点
- **缺点**: 对参数敏感、难以处理不同密度的簇
- **适用场景**: 簇形状不规则、存在噪声的数据

### 4. 高斯混合模型 (Gaussian Mixture Model, GMM)
- **原理**: 假设数据由多个高斯分布混合而成
- **算法**: 使用EM算法估计参数
- **优点**: 可以给出软聚类结果（概率）、更灵活
- **缺点**: 需要预设成分数、可能收敛到局部最优
- **适用场景**: 簇形状为椭圆形的数据

### 5. 谱聚类 (Spectral Clustering)
- **原理**: 基于图论的聚类方法，利用数据的相似度矩阵
- **步骤**:
  1. 构建相似度图
  2. 计算拉普拉斯矩阵
  3. 计算特征向量
  4. 对特征向量进行K-Means聚类
- **优点**: 可以发现任意形状的簇
- **缺点**: 计算复杂度高、对参数敏感
- **适用场景**: 非凸形状的簇

## 评估指标

### 1. 轮廓系数 (Silhouette Score)
```
s = (b - a) / max(a, b)
```
- a: 样本到同簇其他点的平均距离
- b: 样本到最近其他簇所有点的平均距离
- 范围: [-1, 1]，越大越好

### 2. Calinski-Harabasz 指数
- 簇间离散度与簇内离散度的比值
- 越大越好

### 3. Davies-Bouldin 指数
- 簇内距离与簇间距离的比值
- 越小越好

### 4. 调整兰德指数 (Adjusted Rand Index, ARI)
- 需要真实标签
- 衡量聚类结果与真实标签的一致性
- 范围: [-1, 1]，越大越好

## 如何选择K值

### 1. 肘部法则 (Elbow Method)
- 绘制不同K值对应的SSE（误差平方和）
- 选择SSE下降速度开始变缓的点

### 2. 轮廓系数法
- 计算不同K值的平均轮廓系数
- 选择轮廓系数最大的K值

### 3. Gap Statistic
- 比较实际数据的聚类效果与参考分布的差异

## 使用建议

1. **数据预处理**
   - 标准化/归一化
   - 降维（如PCA）以提高效率
   - 处理异常值

2. **算法选择**
   - 簇形状规则：K-Means
   - 簇形状不规则：DBSCAN、谱聚类
   - 需要概率结果：GMM
   - 需要层次结构：层次聚类

3. **参数调优**
   - 使用轮廓系数等指标评估
   - 可视化验证聚类效果
   - 结合领域知识

## 应用场景

- 客户细分
- 图像分割
- 异常检测
- 文档聚类
- 基因表达数据分析
- 推荐系统
