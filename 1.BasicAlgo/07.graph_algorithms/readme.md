# 图算法 (Graph Algorithms)

## 1. 简介

图算法处理由节点和边组成的图结构，是解决网络、路径、关系问题的核心工具。

## 2. 图的表示

### 2.1 邻接矩阵
```
    A B C D
A [ 0 1 1 0 ]
B [ 1 0 1 1 ]
C [ 1 1 0 0 ]
D [ 0 1 0 0 ]
```
- 空间复杂度: O(V²)
- 检查边是否存在: O(1)
- 适合: 稠密图

### 2.2 邻接表
```
A -> [B, C]
B -> [A, C, D]
C -> [A, B]
D -> [B]
```
- 空间复杂度: O(V+E)
- 遍历邻居: O(degree)
- 适合: 稀疏图

## 3. 最短路径算法

### 3.1 Dijkstra算法
**适用**: 非负权重图，单源最短路径

**算法步骤**:
1. 初始化距离数组，源点为0，其余为∞
2. 选择未访问中距离最小的节点
3. 更新其邻居的距离
4. 重复直到所有节点访问完毕

**时间复杂度**: O((V+E) log V) (优先队列)

### 3.2 Bellman-Ford算法
**适用**: 可有负权边，检测负环

**算法步骤**:
1. 初始化距离
2. 对所有边进行V-1次松弛操作
3. 检查是否还有可松弛的边（检测负环）

**时间复杂度**: O(V·E)

### 3.3 Floyd-Warshall算法
**适用**: 所有节点对最短路径

**算法步骤**:
```
for k in range(V):
    for i in range(V):
        for j in range(V):
            dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

**时间复杂度**: O(V³)

### 3.4 算法对比

| 算法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| Dijkstra | O((V+E)logV) | O(V) | 单源，非负权 |
| Bellman-Ford | O(VE) | O(V) | 单源，可有负权 |
| Floyd-Warshall | O(V³) | O(V²) | 所有点对 |

## 4. 最小生成树 (MST)

### 4.1 Kruskal算法
**思想**: 贪心选择最小边，避免形成环

**步骤**:
1. 将所有边按权重排序
2. 依次选择最小的边
3. 如果加入不形成环，则加入
4. 直到有V-1条边

**时间复杂度**: O(E log E)

### 4.2 Prim算法
**思想**: 从一个顶点开始，逐步扩展树

**步骤**:
1. 任选一个顶点开始
2. 选择连接树和非树节点的最小边
3. 将新节点加入树
4. 重复直到所有节点加入

**时间复杂度**: O(E log V) (优先队列)

## 5. 拓扑排序

**适用**: 有向无环图 (DAG)

### 5.1 Kahn算法
```
1. 计算所有节点的入度
2. 将入度为0的节点加入队列
3. 取出队列节点，加入结果
4. 减少其邻居的入度，入度为0则加入队列
5. 重复直到队列为空
```

### 5.2 DFS算法
```
1. 对图进行DFS
2. 节点完成访问时加入栈
3. 最终反转栈得到拓扑序列
```

## 6. 连通性

### 6.1 连通分量 (无向图)
- 使用DFS/BFS遍历
- 每次遍历找到一个连通分量

### 6.2 强连通分量 (有向图)
**Kosaraju算法**:
1. 对原图DFS，记录完成时间
2. 反转所有边
3. 按完成时间降序DFS

**Tarjan算法**: 一次DFS完成

## 7. 网络流

### 7.1 Ford-Fulkerson算法
- 使用增广路径求最大流
- 时间复杂度: O(E·max_flow)

### 7.2 Edmonds-Karp算法
- 使用BFS找最短增广路径
- 时间复杂度: O(V·E²)

## 8. 应用场景

| 算法 | 应用 |
|------|------|
| Dijkstra | 导航系统、网络路由 |
| MST | 网络设计、聚类 |
| 拓扑排序 | 任务调度、编译依赖 |
| 连通分量 | 社交网络分析 |
| 最大流 | 交通流量、资源分配 |
